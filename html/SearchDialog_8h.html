Detailed Description

Pure-UI search dialog — delegates all search logic to FileSearcher.

The dialog owns no search algorithm. It receives a FileSearcher* (injected by SearchController) and connects to its resultFound / finished signals to stream results into the list widget.

This satisfies DIP: SearchDialog depends on the FileSearcher abstraction, not on a concrete search implementation.
